origin/main を main ブランチへ pull rebase で取り込みます。取り込んだコミットをもとに変更内容をまとめて表示し、手動操作が必要な場合は強調表示します。

## 手順

以下のステップを順番に実行してください。

---

### 1. 事前状態の確認

以下のコマンドを並列実行して現在の状態を把握する。

- `git status` — ブランチと作業ツリーの状態
- `git log HEAD..origin/main --oneline` — 取り込まれる予定のコミット一覧（fetch 前の情報）
- `git stash list` — stash の有無

現在のブランチが `main` でない場合はエラーを報告して中断する。
作業ツリーに未コミットの変更がある場合は警告して、続行するか確認する。

---

### 2. fetch して取り込み予定を確認

```bash
git fetch origin
```

fetch 後、`git log HEAD..origin/main --oneline` で取り込まれるコミット数を確認する。
取り込むコミットがゼロの場合は「すでに最新です」と報告して終了する。

---

### 3. pull rebase を実行

```bash
git pull --rebase origin main
```

---

### 4. コンフリクト・エラーのハンドリング

rebase 中にコンフリクトが発生した場合：

> **⚠️ 手動操作が必要です**
>
> rebase がコンフリクトで停止しました。以下の手順で解決してください：
>
> 1. `git status` でコンフリクトファイルを確認
> 2. 各ファイルのコンフリクトマーカー（`<<<<<<<`, `=======`, `>>>>>>>>`）を手動で解決
> 3. `git add <解決したファイル>` でステージング
> 4. `git rebase --continue` で再開
>
> 中断したい場合: `git rebase --abort`

コンフリクトが発生した場合はここで処理を止め、ユーザーに手動解決を依頼する。

---

### 5. 取り込んだ変更のサマリーを表示

pull rebase 成功後、以下のコマンドで変更を調査する。

- `git log ORIG_HEAD..HEAD --oneline` — 取り込んだコミット一覧
- `git diff ORIG_HEAD..HEAD --stat` — 変更ファイルの統計
- `git log ORIG_HEAD..HEAD --format="%h %s%n%b"` — コミット詳細（件名＋本文）

以下の形式でまとめて表示する。

```
## pull rebase 完了

取り込み元: origin/main
取り込んだコミット数: N件

### コミット一覧（古い順）
- <hash> <件名>
- ...

### 変更ファイル一覧
- <ファイルパス>（追加 / 変更 / 削除）
- ...

### 変更内容のまとめ
<コミットの件名・本文をもとに、どのような修正・追加が行われたかを日本語で簡潔にまとめる>
```

取り込んだコミットがない（すでに最新だった）場合は「最新の状態です。取り込むコミットはありませんでした。」と報告する。
