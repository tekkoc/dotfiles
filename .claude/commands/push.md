main ブランチを origin/main へ push します。push 前に内容を確認し、問題がなければ実行します。

## 手順

以下のステップを順番に実行してください。

---

### 1. push 内容の確認

以下のコマンドを並列実行して push 対象を把握する。

- `git status` — 現在のブランチとクリーン状態を確認
- `git log origin/main..HEAD --oneline` — push されるコミット一覧
- `git diff origin/main..HEAD` — push される変更の全差分

現在のブランチが `main` でない場合はエラーを報告して中断する。

---

### 2. 秘匿情報・個人情報のチェック

push 対象の差分を精査し、以下が含まれていないか確認する。

- API キー、トークン、シークレット、パスワード
- 環境変数ファイル（`.env`, `.env.local` など）の実際の値
- 秘密鍵・証明書（PEM, p12 など）
- 個人を特定できる情報（メールアドレス、電話番号、住所など）
- クレデンシャルが埋め込まれた設定ファイル

**問題が見つかった場合は即座に報告して push を中断する。**

---

### 3. push 内容のサマリーをユーザーに提示し、確認を求める

以下の形式で push 内容をまとめて提示する。

```
## push 内容の確認

push 先: origin/main

### コミット一覧（新しい順）
- <hash> <件名>
- ...

### 変更ファイル一覧
- <ファイルパス>（追加 / 変更 / 削除）
- ...

### セキュリティチェック
✅ 秘匿情報・個人情報は検出されませんでした

push しますか？
```

ユーザーが「はい」「yes」「y」「ok」「push」などの確認応答をするまで push しない。
ユーザーが「いいえ」「no」「n」「キャンセル」「中止」などを応答した場合は中断する。

---

### 4. push を実行

```bash
git push origin main
```

---

### 5. 完了報告

push したコミット数とコミット一覧を報告する。
