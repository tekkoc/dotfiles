他の環境の setup.md 相当ファイルをこのリポジトリに取り込んでください。

## インポート元ファイル

$ARGUMENTS

## 手順

### 1. ファイルの読み込みと差分の把握

- インポート元ファイル（上記パス）を読み込む
- このリポジトリの `setup.md` を読み込む
- 両者を比較して、**このリポジトリに存在しない・異なる**内容を特定する
- 差分の概要をリストアップしてから次のステップへ進む

### 2. setup.md の更新

`setup.md` をインポート元の内容に合わせて更新する。

- 既存記述との矛盾・重複がないか確認してから書き換える
- 追加されたツール・設定は適切なセクションに追記（セクションがなければ新設）
- 削除・無効化されていた設定は除去する
- 「この環境はこう動作する」という事実ベースで記述する（手順書ではなく状態の記録）

### 3. dotfiles への反映

差分に応じて、以下のファイルを更新する。各ファイルを読み込んでから編集すること。

#### ツールの追加（Brewfile）
- `homebrew/Brewfile` に `brew "ツール名"` / `cask "ツール名"` を追記
- 既存エントリと重複しないか確認

#### エイリアス・関数の追加・変更
- `zsh/.zsh/aliases.zsh` を更新
- 既存エイリアスとの衝突を確認

#### zshrc の初期化コード
- `zsh/.zshrc` に初期化コード（`eval "$(tool init zsh)"` 等）を追記
- 既に類似コードがある場合はスキップ

#### tmux 設定
- `tmux/.tmux.conf` を更新

#### Ghostty 設定
- `ghostty/config` を更新

#### Neovim プラグイン
- `nvim/lua/plugins.lua` を更新

#### その他
- 差分の内容に応じて適切なファイルを更新する

### 4. CHANGES.md への追記

以下のコマンドで現在日付を取得する：

```bash
date '+%Y-%m-%d'
```

`CHANGES.md` の先頭（既存エントリの上）に以下の形式で追記する：

```
## YYYY-MM-DD

- <変更内容を箇条書きで>
```

同じ日付のエントリが既に存在する場合は、そのエントリに箇条書きを追記する。

### 5. 変更内容の報告

以下の形式で変更内容を報告する：

- **追加されたツール・設定**のリスト
- **変更されたファイル**のリスト
- **スキップした差分**（既に存在・不要と判断したもの）

報告後、手動でのコミット指示を待つ。`/commit` の実行を促す。
