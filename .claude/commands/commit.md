git の差分を確認して、コミットメッセージを考えてコミットしてください。

## 手順

以下のステップを順番に実行してください。

---

### 1. 差分の確認

以下のコマンドを並列実行して現在の状態を把握する。

- `git status` — 変更ファイルの一覧
- `git diff` — unstaged の差分
- `git diff --staged` — staged の差分
- `git log --oneline -5` — 直近のコミット（メッセージスタイルの参考）

---

### 2. 秘匿情報・個人情報のチェック

差分の内容を精査し、以下が含まれていないか確認する。

- APIキー、トークン、シークレット、パスワード
- 環境変数ファイル（`.env`, `.env.local` など）の実際の値
- 秘密鍵・証明書（PEM, p12 など）
- 個人を特定できる情報（メールアドレス、電話番号、住所など）
- クレデンシャルが埋め込まれた設定ファイル

**問題が見つかった場合は即座に報告してコミットを中断する。**

---

### 3. コミットメッセージの作成

差分の内容からコミットメッセージを日本語で作成する。

ルール:
- 1行目（件名）: 変更内容を端的に表す（50文字以内目安）
- 件名は「何をしたか」が伝わる動詞で始める（追加、修正、削除、リファクタ など）
- 複数の変更がある場合は最も重要な変更を件名に、詳細は本文に箇条書きで列挙
- このリポジトリの過去コミットのスタイルに合わせる
- 末尾に必ず以下を付ける:

```
Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
```

---

### 4. ステージングとコミット

- `git add` で関連ファイルをステージング（ファイル名を明示して個別に追加する。`git add .` や `git add -A` は使わない）
- コミットメッセージは HEREDOC 形式で渡す

```bash
git commit -m "$(cat <<'EOF'
件名

- 変更点1
- 変更点2

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>
EOF
)"
```

---

### 5. 完了報告

コミットハッシュとメッセージを報告する。
